# Security Policy · Politica di sicurezza

> 🇬🇧 [English](#english) · 🇮🇹 [Italiano](#italiano)

---

## English

### Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you discover a security issue, contact the maintainer privately:

- 🌐 [federicocalo.dev](https://federicocalo.dev) — see the contact section
- 📧 The address listed on that page

Please include:

- A description of the issue and its potential impact
- Steps to reproduce
- The affected components, versions and configurations
- Any suggested mitigation

We will acknowledge your report within **72 hours** and aim to provide an initial assessment within **7 days**.

### Disclosure policy

We follow a **coordinated disclosure** approach:

1. You report the issue privately
2. We confirm and investigate
3. We develop and test a fix
4. We release the fix and credit you (if you wish) in the changelog
5. After release, the issue may be disclosed publicly

### Scope

In scope:

- The Jarvis core server, agents, frontend, plugins
- Authentication and identity management
- Memory and data storage
- Cryptographic implementations

Out of scope:

- Vulnerabilities in third-party services not deployed by this project
- Social engineering, physical attacks, denial-of-service from public traffic
- Issues already publicly disclosed

### Acknowledgements

Security researchers who responsibly disclose vulnerabilities are gratefully credited in our release notes.

## Italiano

### Segnalare una vulnerabilità

**Non aprire una issue pubblica su GitHub per le vulnerabilità di sicurezza.**

Se scopri un problema di sicurezza, contatta il maintainer in privato:

- 🌐 [federicocalo.dev](https://federicocalo.dev) — vedi la sezione contatti
- 📧 L'indirizzo presente in quella pagina

Includi:

- Una descrizione del problema e dell'impatto potenziale
- I passi per riprodurre
- Componenti, versioni e configurazioni coinvolte
- Eventuali mitigazioni suggerite

Risponderemo alla segnalazione entro **72 ore** e forniremo una prima valutazione entro **7 giorni**.

### Politica di disclosure

Adottiamo un approccio di **disclosure coordinata**:

1. Tu segnali in privato
2. Noi confermiamo e indaghiamo
3. Sviluppiamo e testiamo la fix
4. Rilasciamo la fix e ti accreditiamo (se vuoi) nel changelog
5. Dopo il rilascio, il problema può essere divulgato pubblicamente

### Ambito

Nell'ambito:

- Il server core di Jarvis, gli agenti, il frontend, i plugin
- Autenticazione e gestione dell'identità
- Memoria e storage dei dati
- Implementazioni crittografiche

Fuori ambito:

- Vulnerabilità in servizi di terze parti non deployati dal progetto
- Social engineering, attacchi fisici, denial-of-service da traffico pubblico
- Problemi già divulgati pubblicamente

### Riconoscimenti

I ricercatori di sicurezza che effettuano una disclosure responsabile vengono ringraziati nelle release notes.
